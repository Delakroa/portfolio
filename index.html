Как я пишу автотесты

Архитектура фреймворка
🧪
Test Layer
JUnit 5 тест-классы с аннотациями @TmsLink, @DisplayName. Логика теста — только бизнес-сценарий, без деталей реализации.
PlaywrightSelenideJUnit 5

📄
Page Object Layer
Страницы инкапсулируют локаторы и действия. Тесты не знают про CSS-селекторы — только про методы типа clickApplyFilter().

Паттерны и принципы
🏗️
Page Object Model
Каждый экран — отдельный класс. Тест читается как сценарий, не как набор локаторов. Легко поддерживать при изменении UI.
🔨
Builder Pattern
Генерация тестовых данных через Builders + DataFaker. Каждый тест получает уникальные данные — нет зависимостей между тестами.
⏱️
Log Waiter
Вместо Thread.sleep() — ожидание конкретного события в логах (CUSTOMER, AGENT_ASSIGNED). Тесты стабильны и не падают на флаках.
🔄
Retry Extension
Кастомный JUnit 5 extension для автоматического повтора нестабильных тестов. Настраиваемое количество попыток через аннотацию.
🔗
TMS Integration
Результаты тестов автоматически отправляются в TMS через @TmsLink. После прогона статусы обновляются без ручного труда.
🌐
Multi-Protocol
Один тест может: отправить сообщение через REST, назначить агента через gRPC, проверить статус в Kafka и валидировать данные в БД.
Пример структуры теста
DialogueFilterTest.java 
@ExtendWith({TmsExtension.class})
public class DialogueFilterTest extends BaseApiTest {

    @BeforeEach
    void prepare() {
        dialogues.allDialoguesPage.clickAllDialogues();
        dialogues.dialogueFilterPage.clickCloseFilterIfExists();
    }

    @Test
    @TmsLink("7034")
    @DisplayName("Применить фильтр по каналу WhatsApp")
    void checkChannelFilter() {
        // 1. Подготовить данные — отправить сообщение через API
        var message = builder.message.getRandomSendMessageDTO();
        operations.log().actionAndWaitForLog(
            () -> messageService.sendMessage(message),
            LogTypeList.CUSTOMER.toString()
        );

        // 2. Применить фильтр на UI
        dialogues.dialogueFilterPage.clickDialoguesFilter();
        dialogues.dialogueFilterPage.selectChannelsDropdown();
        dialogues.dialogueFilterPage.selectChannelWhatsApp();
        dialogues.dialogueFilterPage.clickApplyFilter();

        // 3. Проверить результат
        assertThat(dialogues.allDialoguesPage
            .isExistsInFirstRowByProfile(message.getProfileName()))
            .isVisible();
    }
}
